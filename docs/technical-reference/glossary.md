# Glossary

Key terms and concepts in RGB protocol.

## A

### Anchor
Bitcoin transaction that commits to an RGB state transition.

### AluVM
Virtual machine used for RGB contract validation scripts.

### Allocation
Assignment of owned state to a specific seal (UTXO).

## B

### Blinded UTXO
UTXO reference hidden using cryptographic blinding for privacy.

## C

### Client-Side Validation
Validation performed by clients, not miners/validators.

### Consignment
Package containing all data needed to validate an RGB transfer.

### Contract
Instance of an RGB schema with specific state.

### Contractum
Domain-specific language for defining RGB contracts.

## G

### Genesis
Initial state of an RGB contract.

### Global State
State shared across all contract participants.

## I

### Invoice
Payment request encoding recipient, amount, and asset.

## O

### Owned State
State associated with specific UTXOs/owners.

## R

### RGB20
Standard schema for fungible tokens.

### RGB21
Standard schema for non-fungible tokens.

## S

### Schema
Definition of contract structure and rules.

### Seal
Bitcoin UTXO that owns/locks RGB state.

### Single-Use-Seal
UTXO that can be spent only once, ensuring state progression.

### Stash
Local storage for RGB contract state and history.

### State Transition
Operation that changes contract state.

### Strict Encoding
Deterministic binary encoding system.

## T

### Taproot
Bitcoin upgrade used for RGB commitments.

### Transition
See State Transition.
